# Repository Guidelines

Python Slack bot built on OpenAI's **Responses API** (never Chat Completions). The
design is stateless: Slack is the source of truth, and conversation context is
rebuilt from platform history on demand. See `CLAUDE.md` for the deeper
architecture notes.

## Project Structure & Module Organization
- Entry points: `main.py` (platform selection), `slackbot.py` (Slack runner),
  `settings_modal.py` (settings modal UX).
- Core functionality lives in **packages**, not single modules:
  - `message_processor/` — message pipeline, handlers, tools, participation logic
  - `openai_client/` — Responses API wrapper (`api/`, `base.py`, `utilities.py`)
  - `slack_client/` — Slack transport, `event_handlers/`, `formatting/`
  - `streaming/` — streaming buffer, fence handling, rate limiting, native sink
- Root modules hold shared pieces: `config.py`, `database.py`, `thread_manager.py`,
  `document_handler.py`, `prompts.py`, `markdown_converter.py`, `tool_registry.py`,
  `mcp_manager.py`, `logger.py`, `token_counter.py`, `base_client.py`.
- `Docs/` holds specs and plans. Runtime-only dirs: `data/` (SQLite), `logs/`.
- Tests split by scope under `tests/unit/` and `tests/integration/`; shared fixtures
  in `tests/conftest.py`. **Test files go in `tests/`, never the repo root.**

## Build, Test, and Development Commands
- `make install` — install the pinned, hashed lockfile (`pip install --require-hashes -r requirements.txt`).
- `make test` — unit suite (`tests/unit`) with terminal + HTML coverage.
- `make test-unit` / `make test-fast` — unit tests, without the coverage run.
- `make test-integration` / `make test-all` — integration scenarios; these hit real APIs.
- `make lint` — `ruff check .` then `mypy .` (each skipped if not installed).
- `make format` — `black .` then `isort .` (each skipped if not installed).
- `make check` — `lint` + `test`.
- `python3 slackbot.py` — launch the Slack bot once the environment is configured.

## Dependencies
- `requirements.in` is the human-edited source of truth; `requirements.txt` is an
  autogenerated lockfile with exact pins + sha256 hashes. **Never hand-edit the lockfile.**
- Add/remove a dep: edit `requirements.in`, run `make lock`, commit both files.
  `make lock-upgrade` refreshes pins within the existing constraints.
- System packages the bot genuinely shells out to (all via `document_handler.py`):
  `poppler-utils` (PDF page rendering), `tesseract-ocr` (OCR of scanned PDFs), and
  `pandoc` (DOCX fallback extraction). All degrade gracefully when absent.

## Coding Style & Naming Conventions
- Python 3.12 (the lockfile is compiled for 3.12). Four-space indent, `snake_case`
  for modules/functions/variables, `PascalCase` for classes.
- Prefer lean, single-responsibility modules; extract helpers into adjacent files
  rather than growing the runners.
- Format with `black` + `isort`, lint with `ruff`, keep type hints passing `mypy`.

## Testing Guidelines
- Name test files `test_*.py` (see `pytest.ini`). Markers: `unit`, `integration`,
  `critical`, `smoke` — `--strict-markers` is on, so register any new marker.
- Unit tests must not touch the network. Integration tests use real API keys from
  `.env` and are opt-in via `make test-integration` / `make test-all`.
- Never start the bot as part of a test run; the maintainer manages the running process.

## Commit & Pull Request Guidelines
- History uses **plain imperative subjects — no conventional-commit prefixes**
  (e.g. "Fix: forward tool_event_callback through the OpenAIClient wrapper",
  "Expose link previews as ENABLE_LINK_PREVIEWS (default off)"). Match that; do not
  introduce `feat:` / `refactor:` / `docs:`.
- Keep the subject short and imperative; put detail in the body.
- Update `CHANGELOG.md` alongside user-visible changes.
- PRs should state intent and list validation steps (`make test`, `make lint`). There
  is no CI workflow in this repo — only Dependabot — so run the checks locally.

## Configuration & Secrets
- Copy `.env.example` when onboarding; never commit secrets. Required keys are
  **`OPENAI_KEY`** (not `OPENAI_API_KEY`), plus `SLACK_BOT_TOKEN` and `SLACK_APP_TOKEN`.
- Use `slack_app_manifest.example.yml` as the template for scopes and event
  subscriptions; `slack_app_manifest.yml` is gitignored and environment-specific.
- If `.venv/` exists, activate it before running Python commands:
  `source .venv/bin/activate`.
